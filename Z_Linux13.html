부러의 다락방


삼바는 - 리눅스 + 윈도우 서버 연결
NFS는 - 리눅스 + 리눅스 서버 연결

672쪽
root@smb

서버 NFS(새로운거)하고 SAMVA(메일서버였던것)켜기

호스트전용어댑터
삼바 = 192.168.56.102
엔에프에스 = 192.168.56.104


NFS = 네트워크 붙이는 거
리눅스 서버 연결하는 거

스토리지 서버가 있으면
하드 해서 거따가 붙임

NFS의 하드를 SAMBA에 달기
어따 넣든 둘 중 하나는 서버 돌아가야 함

(NFS) (NAT인 상태로 설치)
sudo apt -y install nfs-kerner-server

장치 네트워크 > 네트워크설정 > 호스트전용어댑터로 변경
sudo addr 확인 이번엔 104

sudo nano /etc/exports 
/nfs		192.168.56.102(rm,sync)
- 접근하도록 설정 (상대 머신 아이피는 알아서 확인)

mkdir /nfs
chmod 707 /nfs
systemctl restart nfs-server
systemctl enable nfs-server

exports -v
> 뭐 되는지 /nfs 192.168.56.102 뜸

(이제 삼바서버수정 NAT 으로 변경 )

sudo apt -y install nfs-common

( 이제 호스트 전용어댑터로 변경 )

showmount -e 192.168.56.104 //붙일 수 있는 거 뭐냐?
> /nfs 192.168.56.104 뜸

mount -f nfs 192.168.56.104:/nfs /nfs

// nfs를 nfs에다가 붙여라 하지만 경로 없어서 오류남

mkdir /nfs
