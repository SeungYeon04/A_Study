부러의 다락방


  • Parity Check

    • Parity Check > (에러가 있는지만 검사 가능)
    ⭐ 전체 1의 갯수가 중점이다.

    1011[1] 짝수 프리티비트(우수odd)
    1011[0] 홀수 프리티비트(기수even)

    송신 --- 수신
    10111 > 10111 에러X
    10111 > 10101 에러o

    짝수로 보낸다고 하고
    홀수가 오면 오류임을 알 수 있음.
    그런데 두 비트나 달라지면
    10001 에러로 판단을 할 수가 없음.

  • 수평수직 Parity bit

    • 위 아래 다 짝수 맞추고 다 맞으면 오케이다.
    짝수 보내면 위아래 다 짝수로
    10100
    10001
    01010
    01111

    단점: 데이터가 많아서 느리다.(오버헤드 있다)
    괜히 데이터 더 붙이는 식이라 효과는 그닥!.

    교수님이 사사오임 = 반올림 같은 말인 거
    세대 다르니까 알고만 있으라캄

  • Hamming Code(Smgle error Correecting code)

    • 단일에러 정정 코드 (수정, 교정)

    페리티값
    1 2 3 4 5 6 7
    [][] 1 [] 1 0 0

    ⭐하밍코드 계산공식 순서
    1357: [] 110
    2367: [] 100
    4567: [] 100

    짝수로 판단하는 기준
    1357: 0 110
    2367: 1 100
    4567: 1 100

    1 2 3 4 5 6 7
    0 1 1 1 1 0 0

    0111100모두 짝수인지 확인(맞으면 0 틀리면 1)
    1357: 0 110 > 0
    2367: 1 100 > 0
    4567: 1 100 > 0

    Hamming Code의 오류버전도 보자
    1 2 3 4 5 6 7
    [0][1]1[1]000
    1357: 0100 > 1
    2367: 1100 > 0
    4567: 1000 > 1

    (101)2 = (5)2
    (5)2 부분이 오류라는 것이다.

    Hamming Code의 홀수버전도 보자.
    1 2 3 4 5 6 7
    [1][0]1[1]011

    1357: 1101
    2367: 0111
    4567: 1011

    Hamming Code의 홀수 오류버전도 보자.
    1 2 3 4 5 6 7
    1 0 1 1 0 1 1

    1357: 1101
    2367: 0111
    4567: 1011
    (000)2 = (0)10

    Hamming Code 짝수 오류버전 또 다시
    1 0 1 1 0 0 1
    1357: 1101
    2367: 0101
    4567: 1001
    (110)2 = (6)10


  • GreayCode(1bit씩 변화)

    • 상태가 변하면 1비트씩 바뀌는 것이다.
    맨 왼쪽에 1올라가면 그대로 보존되고
    자릿수 올라갈 때 인접한 두 비트를 비교해서
    값이 다르면 0 아니면 1로 쓴다.
